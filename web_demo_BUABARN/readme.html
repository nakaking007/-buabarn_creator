============================================================
1. ระบบนี้คืออะไร
============================================================

Buabarn VIP Creater Tools คือเครื่องมือช่วยผลิตคอนเทนต์ AI ภาษาไทย
สำหรับสร้างงานพร้อมใช้ ไม่ใช่แค่คำแนะนำ โดยระบบช่วยจัดทำ:

- พร้อมท์ภาพรายฉาก
- พร้อมท์วิดีโอรายฉาก
- Storyboard / โครงเรื่อง
- Voice Over / บทบรรยายเสียง
- Sound Design / เสียงประกอบวิดีโอ
- Caption / ข้อความโพสต์
- เนื้อเพลงไทยและ Suno Style Prompt
- งานขายสินค้า ละครปักตะกร้า รีวิวสินค้า และคอนเทนต์ลง TikTok / YouTube

ระบบออกแบบให้ผู้ใช้พิมพ์ไอเดียเป็นภาษาไทย แล้วได้ชิ้นงานที่แก้ไข คัดลอก
และนำไปใช้ต่อกับ AI สร้างภาพ วิดีโอ เสียง หรือเพลงได้ทันที

============================================================
2. เหมาะกับใคร
============================================================

เครื่องมือนี้เหมาะสำหรับ:

- ผู้สร้างคอนเทนต์ TikTok, YouTube Shorts, Facebook Reels
- ผู้ขายสินค้าออนไลน์ที่ต้องการทำคลิปรีวิวหรือละครปักตะกร้า
- ผู้ทำเพจธรรมะ เรื่องเล่าลึกลับ ความเชื่อไทย และพญานาค
- ผู้ต้องการสร้างภาพ AI คุณภาพสูงเป็นฉาก ๆ
- ผู้ต้องการสร้าง Storyboard สำหรับหนังสั้น
- ผู้ต้องการทำเพลงด้วย Suno โดยมีเนื้อเพลงไทยและ style prompt พร้อมใช้
- ผู้เริ่มต้นที่ยังเขียน prompt ไม่เก่ง แต่ต้องการผลงานที่ดูเป็นระบบ

============================================================
3. ระบบทำอะไรได้บ้าง
============================================================

ระบบใช้ข้อกำหนด output 2 ชั้น:

1. มาตรฐานกลาง: ใช้กับทุกงาน เช่น ชื่อผลงาน, Core Concept, กลุ่มเป้าหมาย, Mood & Tone, Story Structure, Storyboard, Prompt, Negative Prompt และ Quality Checklist
2. มาตรฐานเฉพาะหมวด: แต่ละเครื่องมือมี output ต่างกัน เช่น เพลงต้องมี BPM/Key/Lyrics, งานขายต้องมี Hook/Pain Point/Offer/CTA, หนังสั้นต้องมี Timeline/Shot List, พญานาคต้องมี Naga Identity Lock

ไฟล์ข้อกำหนด:
- production-specs.js ใช้กำกับระบบจริง
- PRODUCTION_OUTPUT_SPECS.md ใช้อ่านเป็นเอกสารมาตรฐาน
- production-input-specs.js ใช้สร้างช่อง input เฉพาะแต่ละหมวด
- PRODUCTION_INPUT_SPECS.md ใช้อ่านข้อกำหนดช่อง input ของระบบ

3.1 งานภาพ

ระบบสามารถสร้างพร้อมท์ภาพรายฉาก เช่น:

- Prompt ภาพฉากที่ 1
- Prompt ภาพฉากที่ 2
- Prompt ภาพฉากที่ 3

จำนวน prompt จะตรงกับจำนวนฉากที่ผู้ใช้เลือก เช่น 3, 5, 7 หรือ 10 ฉาก
ถ้าผู้ใช้เลือก “ไม่เลือกภาพ” ระบบจะไม่สร้าง prompt ภาพ

3.2 งานวิดีโอ

ระบบสามารถสร้างพร้อมท์วิดีโอรายฉาก พร้อมเสียงประกอบ เช่น:

- Prompt วิดีโอฉากที่ 1
- เสียงประกอบวิดีโอฉากที่ 1
- Voice Over ฉากที่ 1

และทำต่อเนื่องตามจำนวนฉากที่เลือก
ถ้าผู้ใช้เลือก “ไม่เลือกวิดีโอ” ระบบจะไม่สร้าง prompt วิดีโอหรือเสียงประกอบ

3.3 งานเพลง

เมนู “เพลงสำหรับ Suno” แยกออกจากภาพและวิดีโอ เพื่อไม่ให้สับสน
ระบบจะสร้าง:

- Concept เพลง
- Suno Style Prompt
- เนื้อเพลงไทย
- Prompt ปกเพลง
- Caption สำหรับโพสต์

3.4 งานขายสินค้า

ระบบช่วยวางโครงงานขาย เช่น:

- Hook เปิดคลิป
- Pain Point
- จุดขาย
- คำพูดปักตะกร้า
- CTA
- Caption

3.5 วิดีโอ Local

ระบบมีโมดูลเสริมชื่อ BUABARN_LOCAL_VIDEO สำหรับสร้างวิดีโอ MP4 แนวตั้งจากภาพนิ่ง
เหมาะกับ TikTok, Reels และ YouTube Shorts
โมดูลนี้ทำงานผ่าน Python + ffmpeg.exe ในเครื่องผู้ใช้
ไม่ใช่ AI สร้างวิดีโอจากศูนย์ แต่ช่วยให้มีไฟล์วิดีโอจริงจากภาพ + เสียง/สคริปต์ได้

============================================================
5A. การติดตั้งและการอนุญาตสิทธิ์
============================================================

ในโฟลเดอร์นี้มีไฟล์ install-helper.bat สำหรับช่วยเปิดหน้า Extensions
ผู้ใช้สามารถดับเบิลคลิกไฟล์นี้ แล้วตอบ Y เพื่อเปิดหน้า chrome://extensions ได้

สิ่งที่ควรมีตอนติดตั้ง:

จำเป็น:
- Google Chrome สำหรับติดตั้ง Extension
- Python สำหรับใช้โมดูล BUABARN_LOCAL_VIDEO
- ต้องแตกไฟล์ ZIP ก่อนใช้งาน ห้ามรันจากใน ZIP

มีให้ในแพ็กแล้ว:
- ffmpeg.exe อยู่ในโฟลเดอร์ BUABARN_LOCAL_VIDEO แล้ว ไม่ต้องติดตั้งเพิ่ม
- install-helper.bat สำหรับช่วยเปิดหน้า Chrome Extensions
- run_local_video.bat สำหรับรันโมดูลวิดีโอ local

ติดตั้งเพิ่มเฉพาะกรณีต้องการเสียงไทยอัตโนมัติ:
- gTTS ใช้คำสั่ง python -m pip install gTTS
- จากนั้นเปิด BUABARN_LOCAL_VIDEO/config.json แล้วเปลี่ยน "use_gtts": false เป็น "use_gtts": true

ถ้าต้องการสร้างภาพจริง:
- ต้องมี Hugging Face Token หรือ API ภาพอื่น ๆ
- ลิงก์รับ API อยู่ในเมนูกรอก API

ถ้าต้องการสร้างวิดีโอ AI จากศูนย์:
- ต้องมี API วิดีโอจากผู้ให้บริการภายนอก เช่น fal.ai, Runway หรือ Replicate
- บริการเหล่านี้มักใช้เครดิต ไม่ควรบอกลูกค้าว่าฟรีถาวร

หมายเหตุสำคัญ:

ไฟล์ .bat ไม่สามารถกดอนุญาตสิทธิ์ Chrome แทนผู้ใช้ได้
Chrome บังคับให้ผู้ใช้เป็นคนยืนยันสิทธิ์เอง เพื่อความปลอดภัย

สิทธิ์ที่ระบบอาจขอใช้:

- storage: ใช้บันทึก API/Token และการตั้งค่าในเครื่องผู้ใช้
- clipboardRead / clipboardWrite: ใช้วาง API และคัดลอกชิ้นงาน
- activeTab / scripting: ใช้อ่านชื่อหน้าเว็บและคำอธิบาย เมื่อผู้ใช้กดปุ่มอ่านข้อมูลเว็บ
- host permissions: ใช้เรียก API ภายนอก เช่น Hugging Face
- sidePanel: ใช้เปิดระบบใน Chrome Side Panel

วิธีติดตั้งแบบไฟล์:

1. แตกไฟล์ ZIP ก่อน
2. ดับเบิลคลิก install-helper.bat หรือเปิด Chrome แล้วเข้า chrome://extensions เอง
3. เปิด Developer mode
4. กด Load unpacked
5. เลือกโฟลเดอร์ BUABARN_VIP หรือ release_BUABARN_VIP
6. ถ้า Chrome แสดงคำขอสิทธิ์ ให้ผู้ใช้ตรวจดูและกดยืนยันเอง

ถ้าจะสร้างภาพ/วิดีโอจริง:

- ต้องกรอก API สำหรับสร้างภาพหรือวิดีโอ
- อินเทอร์เน็ตต้องเข้า huggingface.co และ api-inference.huggingface.co ได้
- จากการทดสอบจริง Token เดียวกันสร้างภาพนิ่งได้แล้ว แต่ endpoint วิดีโอของ Hugging Face Provider ยังตอบว่าโมเดลวิดีโอไม่รองรับใน provider ที่เรียกอยู่
- ถ้าต้องการสร้างวิดีโอจริงแบบเสถียร ควรใช้ API วิดีโอเฉพาะทาง เช่น fal.ai, Replicate, Runway, Pika หรือผู้ให้บริการที่มี endpoint วิดีโอพร้อมใช้งาน

ลิงก์รับ API / ทดลองใช้งาน:

API สร้างภาพ
1. Hugging Face: https://huggingface.co/settings/tokens
   เหมาะกับเริ่มต้นและทดสอบภาพนิ่ง
2. fal.ai: https://fal.ai/dashboard/keys
   เหมาะกับงานภาพและวิดีโอหลายโมเดล แต่ควรตรวจเครดิตก่อนใช้งานจริง
3. Replicate: https://replicate.com/account/api-tokens
   เหมาะกับผู้ใช้ที่ต้องการเลือกโมเดลจากหลายเจ้า

API สร้างวิดีโอ
1. fal.ai: https://fal.ai/dashboard/keys
   แนะนำเป็นอันดับแรกสำหรับเชื่อมระบบ เพราะมีโมเดลวิดีโอหลายตัว
2. Runway: https://dev.runwayml.com
   เหมาะกับวิดีโอคุณภาพสูง แต่ต้องใช้เครดิต
3. Replicate: https://replicate.com/account/api-tokens
   เหมาะกับทดสอบโมเดลวิดีโอหลายแบบ

คำเตือนเรื่องความน่าเชื่อถือ:
บริการวิดีโอ AI ส่วนใหญ่ไม่ใช่ฟรีถาวรสำหรับ API จริง บางบริการมีเครดิตฟรีหรือ free tier เฉพาะหน้าเว็บเท่านั้น ก่อนขายงานควรทดสอบคีย์จริงและแจ้งลูกค้าว่าการสร้างวิดีโอใช้เครดิตของผู้ให้บริการภายนอก

============================================================
8. ปุ่ม “อ่านชื่อ/คำอธิบายจากแท็บเว็บ” คืออะไร
============================================================

ปุ่มนี้ใช้ดึงข้อมูลจากหน้าเว็บที่เปิดอยู่ในแท็บปัจจุบัน เช่น:

- ชื่อหน้าเว็บ
- คำอธิบายเว็บไซต์

เพื่อช่วยกรอกไอเดียเบื้องต้นให้เร็วขึ้น

หมายเหตุ:
บางเว็บไซต์ไม่อนุญาตให้ส่วนขยายอ่านข้อมูล
ถ้าดึงไม่ได้ ให้พิมพ์ไอเดียเองในช่องคำสั่ง

============================================================
9. คำแนะนำให้ได้งานคุณภาพสูง
============================================================

- เขียนไอเดียให้ชัดว่า ใคร ทำอะไร ที่ไหน เวลาใด อารมณ์แบบไหน
- ใส่รายละเอียดเครื่องแต่งกาย แสง สี ฉาก และวัตถุสำคัญ
- ถ้ามีตัวละครหรือสินค้า ควรแนบภาพอ้างอิงให้ครบ 3 ภาพ
- เลือกจำนวนฉากให้เหมาะกับงาน
- งานภาพควรเริ่มที่ 3 ฉากก่อน
- งานวิดีโอควรใช้ 3-5 ฉากเพื่อให้จังหวะไม่ยาวเกินไป
- งานเพลงควรแยกทำในเมนูเพลงสำหรับ Suno
